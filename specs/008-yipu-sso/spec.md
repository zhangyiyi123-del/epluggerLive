# Feature Specification: 易普圈与我的易普 SSO 对接

**Feature Branch**: `008-yipu-sso`  
**Created**: 2026-04-01  
**Status**: Draft  
**Input**: User description: "结合《我的易普 SSO 对接指南》，易普圈与我的易普（epWorkApp）对接，通过我的易普获取用户信息并完成登录。"

> 说明：与 `specs/007-yipu-sso/spec.md` 需求一致；本目录为 `/speckit.plan` 当前选中的规格路径。

## 背景与对接要点（来自对接方指南的抽象）

- **发起方**：用户在「我的易普」应用内通过「外部系统」入口进入易普圈；发起方仅向白名单中的目标地址签发跳转。
- **信任载体**：短时效、带签名的令牌；令牌中包含用户唯一标识与画像字段（姓名、岗位/角色、邮箱、手机号、头像链接等），以及签发方标识、时间戳、过期时间与一次性随机串。
- **易普圈侧**：在落地环节校验令牌真实性、有效期与一次性使用，据此建立易普圈内的已登录身份，并让用户在易普圈内继续使用功能（含获取「当前用户」信息）。

本规格描述易普圈**应具备的能力与验收标准**，不约定具体编程语言或中间件实现方式。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 从「我的易普」免重复登录进入易普圈 (Priority: P1)

已登录「我的易普」的用户从应用内入口跳转到易普圈；在令牌有效且未被篡改的前提下，用户应在易普圈内自动成为已登录状态，无需再次输入账号密码。

**Why this priority**: 这是对接的核心价值：统一入口与身份，降低摩擦并保证与发起方身份一致。

**Independent Test**: 在联调环境完成一次完整跳转后，仅验证「落地后易普圈显示为已登录且可拉取当前用户资料」即可独立证明本故事价值。

**Acceptance Scenarios**:

1. **Given** 用户在「我的易普」中已登录且发起方已为易普圈配置白名单与目标地址，**When** 用户选择进入易普圈并完成跳转，**Then** 易普圈应建立该用户的已登录会话并进入可用状态。
2. **Given** 跳转所带令牌在允许的时间窗口内且未被重放，**When** 用户完成落地，**Then** 易普圈应拒绝同一令牌的第二次使用。

---

### User Story 2 - 易普圈内使用与发起方一致的用户画像 (Priority: P2)

登录成功后，易普圈在展示或业务逻辑中需要的用户标识与画像字段（姓名、岗位/角色、邮箱、手机号、头像）应与发起方令牌中传递的内容一致（空值按「无此信息」处理，不因缺字段而崩溃）。

**Why this priority**: 对接不仅是「能登录」，还要让圈内体验与主应用身份信息对齐。

**Independent Test**: 在已知测试账号画像的前提下，登录后查看「当前用户」或等价能力，比对字段是否与发起方配置一致。

**Acceptance Scenarios**:

1. **Given** 发起方在令牌中提供了完整画像字段，**When** 用户在易普圈内查看个人相关展示或调用「当前用户」能力，**Then** 应展示与令牌一致的姓名、岗位/角色、邮箱、手机号、头像（若有）。
2. **Given** 发起方某画像字段为空字符串，**When** 用户使用易普圈，**Then** 易普圈应优雅处理缺失项（如不展示或显示占位），且不应因该字段为空而失败。

---

### User Story 3 - 非法或失效跳转被拒绝并安全失败 (Priority: P2)

篡改、过期、格式错误或已被使用过的令牌不得建立登录态；目标地址未在发起方白名单时不应由发起方成功签发（由发起方负责），易普圈侧对无效落地请求应明确失败且不泄露敏感细节。

**Why this priority**: 安全与可运维性是上线必要条件。

**Independent Test**: 分别构造过期令牌、篡改令牌、重复令牌三种用例，验证均无法获得已登录会话。

**Acceptance Scenarios**:

1. **Given** 令牌已过期，**When** 用户或攻击者使用该令牌访问落地入口，**Then** 易普圈不得建立登录态并应给出用户或运维可理解的失败结果（不含完整令牌内容）。
2. **Given** 令牌签名或内容被篡改，**When** 访问落地入口，**Then** 易普圈应拒绝登录。
3. **Given** 同一令牌已被成功消费过一次，**When** 再次使用该令牌，**Then** 易普圈应拒绝第二次建立会话。

---

### User Story 4 - 运维与密钥管理可落地 (Priority: P3)

生产环境应通过安全方式配置与「我的易普」共用的签名密钥；日志与排障规范满足「不输出完整令牌、可追踪失败原因与用户标识片段」的运维要求。

**Why this priority**: 满足上线检查与长期运维，但不阻塞核心登录路径的首次联调。

**Independent Test**: 审查配置与日志样例是否符合「密钥不进代码仓库、日志脱敏」的约定。

**Acceptance Scenarios**:

1. **Given** 生产部署，**When** 运维检查配置，**Then** 共享密钥应通过环境变量或组织认可的密钥注入方式提供，而非明文写入版本库。
2. **Given** 发生验签或过期失败，**When** 查看审计或错误日志，**Then** 不应出现完整令牌明文，且可关联到失败类型与必要的用户标识前缀（若适用）。

### Edge Cases

- 发起方配置的跳转目标与易普圈实际环境域名不一致（如多环境、CDN）时，白名单与目标 URL 需双方对齐，否则用户无法完成跳转（属配置问题，易普圈应提供明确的环境与域名说明给对接方）。
- 同一自然人在发起方侧 `uid` 不变但岗位等信息变更：再次跳转登录后，易普圈内展示或存储的用户信息应更新为最新令牌内容（在易普圈产品策略允许的前提下）。
- 首次到达的 `uid` 在易普圈无历史记录：默认采用自动创建圈内用户档案并绑定该 `uid`；若产品策略要求「仅允许预注册用户使用」，则拒绝登录并给出可理解的提示（见假设）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 易普圈 MUST 提供从「我的易普」跳转后的落地入口，用于接收短时效信任令牌并完成身份校验。
- **FR-002**: 易普圈 MUST 使用与「我的易普」事先约定的共享密钥，校验令牌签名与完整性；校验失败不得建立已登录状态。
- **FR-003**: 易普圈 MUST 校验令牌是否在有效期内；过期令牌不得建立会话。
- **FR-004**: 易普圈 MUST 对令牌中的随机串（一次性因子）做消费记录，防止同一令牌被重复使用（重放）。
- **FR-005**: 易普圈 MUST 在校验通过后，将令牌中的用户唯一标识与画像字段（姓名、岗位/角色、邮箱、手机号、头像链接；空值以空字符串形式出现）映射为圈内用户身份；邮箱、手机号、头像等若无值须按空值处理且不得省略字段校验逻辑导致异常。
- **FR-006**: 易普圈 MUST 在成功校验后为该用户建立易普圈内的已登录状态，使用户可继续访问需登录的功能。
- **FR-007**: 易普圈 MUST 提供「当前登录用户」信息供前端或业务使用，其内容与最近一次成功落地所验证的身份一致。
- **FR-008**: 易普圈 SHOULD 在落地成功后以不长期暴露信任令牌的方式将登录结果交接给前端，由前端获得圈内已授权访问能力。
- **FR-009**: 易普圈 SHOULD 在上线环境中限制仅可信网络来源可调用落地入口，以降低未授权批量探测风险（具体网段由双方运维约定）。
- **FR-010**: 易普圈 MUST 在日志与排障信息中对令牌脱敏（例如仅保留前缀若干字符），禁止输出完整令牌或共享密钥。

### Key Entities

- **SSO 信任令牌（概念）**: 由「我的易普」签发、易普圈消费的短时效、带签名载体；包含用户标识、画像字段、时间、过期、一次性随机串、签发方标识等声明。
- **圈内用户身份**: 与发起方 `uid` 绑定的易普圈用户档案；用于会话与权限；可随每次成功跳转更新画像字段。
- **一次性因子消费记录**: 用于记录某随机串是否已使用，支撑防重放；生命周期与令牌有效期对齐。

## Assumptions

- 「我的易普」侧将按对接指南配置白名单、目标地址、密钥与系统编码；易普圈提供准确的落地 URL 与运行环境主机名供对方配置。
- 令牌字段集合与签名算法与《我的易普 SSO 对接指南》一致；易普圈实现须与该指南语义兼容（不要求本规格重复字节级协议，技术计划在实现阶段引用指南）。
- 首次通过某 `uid` 进入易普圈时，默认**自动**创建或关联圈内用户档案；若产品后续要求「仅白名单用户可入」，将单独变更规格与验收场景。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在联调环境中，使用有效跳转的测试用户可在 **1 分钟内**完成从「我的易普」到易普圈可用首页的路径，且全程无需在易普圈再次输入密码。
- **SC-002**: 安全用例（过期、篡改、重放）在测试中 **100%** 被拒绝建立已登录会话。
- **SC-003**: 对具备完整画像的测试账号，易普圈「当前用户」信息与发起方传递内容 **字段级一致**（含空字段的约定处理）。
- **SC-004**: 生产就绪检查中，共享密钥 **不得**以明文形式出现在版本库或静态配置提交物中；抽样日志 **不得**出现完整令牌。

## Out of Scope

- 「我的易普」应用内部菜单、列表接口与签发跳转的后台实现（由对方系统负责）。
- 易普圈内除「登录与身份映射」以外的业务功能改造，除非显式依赖当前用户身份（本规格仅覆盖身份对接）。
