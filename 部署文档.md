# EPlugger 上线部署手册（逐步操作版）

> 适用读者：第一次部署本项目的开发者。  
> 部署方案：**同一台 Linux 服务器**，Nginx 做反向代理（用户访问 `http://你的IP 或域名`，Nginx 分发前端页面与后端接口）。  
> 全程无需修改任何源代码。

---

## 准备工作：你需要什么？

| 需要 | 说明 |
|------|------|
| 一台 Linux 服务器 | 推荐 Ubuntu 20.04/22.04 或 CentOS 7/8，内存 ≥ 2GB |
| MySQL 8.x | 安装在服务器上，或能从服务器连通的独立数据库 |
| JDK 17 | 运行后端 JAR |
| Node.js 18 LTS | 只在**构建前端**时需要，构建完可卸载 |
| Maven 3.8+ | 只在**构建后端**时需要，构建完可卸载（也可在开发机构建好再上传 JAR） |
| Nginx | 做反向代理与静态资源托管 |

## 第一步：在服务器上准备 MySQL 数据库

SSH 登录服务器，执行以下操作（如果 MySQL 已装好，跳过安装部分）。

### 1-1. 安装 MySQL（Ubuntu 示例）

```bash
sudo apt update
sudo apt install -y mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql
```

### 1-2. 创建业务数据库和账号

```bash
sudo mysql
```

进入 MySQL 后执行：

```sql
-- 创建数据库，名字固定为 eplugger
CREATE DATABASE eplugger CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建专用账号（用户名/密码自己定，这里示例为 eplugger_user / 你的强密码）
CREATE USER 'eplugger_user'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON eplugger.* TO 'eplugger_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

> 数据库表结构不需要手动建，后端启动时 **Flyway 会自动创建所有表**。

## 第二步：安装 JDK 17

```bash
sudo apt install -y openjdk-17-jdk
java -version   # 应显示 17.x
```

## 第三步：构建后端 JAR

> 这步可以在**开发机**（你的 Windows 电脑）上做，构建好再把 JAR 上传到服务器。也可以在服务器上安装 Maven 后直接做。

### 3-1. 在开发机构建（推荐）

在 `backend` 目录下打开命令行：

```bash
cd backend
mvn -DskipTests package
```

等待构建完成，产物在：

```
backend/target/eplugger-backend-0.1.0-SNAPSHOT.jar
```

### 3-2. 把 JAR 上传到服务器

```bash
# 在开发机执行，将 JAR 上传到服务器的 /opt/eplugger/ 目录
scp backend/target/eplugger-backend-0.1.0-SNAPSHOT.jar 你的用户名@服务器IP:/opt/eplugger/
```

先在服务器上创建该目录：

```bash
sudo mkdir -p /opt/eplugger
sudo mkdir -p /var/lib/eplugger/upload   # 用户上传的文件存这里
```

## 第四步：配置并启动后端

### 4-1. 创建启动脚本

在服务器上创建 `/opt/eplugger/start.sh`：

```bash
sudo nano /opt/eplugger/start.sh
```

粘贴以下内容，**把括号内容替换为你的真实值**：

```bash
#!/bin/bash

# ===== 业务数据库（第一步创建的那个）=====
export EPLUGGER_BUSINESS_DB_URL='jdbc:mysql://localhost:3306/eplugger?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai'
export EPLUGGER_BUSINESS_DB_USERNAME='eplugger_user'
export EPLUGGER_BUSINESS_DB_PASSWORD='你的强密码'

# ===== JWT 密钥（随便写一串至少32个字符的字符串，保密）=====
export JWT_SECRET='请替换成你的随机字符串至少32字符长度abc123xyz'

# ===== 你的域名或服务器公网IP（浏览器访问的地址）=====
# 例如：http://192.168.1.100  或  https://app.example.com
export CORS_ALLOWED_ORIGINS='http://你的域名或IP'
export SSO_FRONTEND_CALLBACK_URL='http://你的域名或IP/sso/callback'

# ===== epWork 外链验签密钥（与 epWork 侧保持一致）=====
export EPWORK_SSO_SECRET='与epWork对端一致的密钥'

# ===== 上传文件存放位置 =====
export UPLOAD_DIR='/var/lib/eplugger/upload'

# ===== 如果暂时不接入易普人员库，保持 false；接入则改为 true 并填写下方配置 =====
export EPWORK_PERSONNEL_SYNC_ENABLED='false'

# 接入人员库时取消注释并填写：
# export EPWORK_PERSONNEL_DB_URL='jdbc:mysql://人员库IP:3306/ep_work_test?useUnicode=true&characterEncoding=UTF-8&serverTimezone=GMT%2b8'
# export EPWORK_PERSONNEL_DB_USERNAME='数据库用户名'
# export EPWORK_PERSONNEL_DB_PASSWORD='数据库密码'

# ===== 启动 =====
java -jar /opt/eplugger/eplugger-backend-0.1.0-SNAPSHOT.jar
```

保存退出（Ctrl+O → Enter → Ctrl+X），然后赋予执行权限：

```bash
sudo chmod +x /opt/eplugger/start.sh
```

### 4-2. 测试启动后端

```bash
sudo bash /opt/eplugger/start.sh
```

看到类似以下日志说明启动成功：

```
Started EpluggerApplication in x.xx seconds
```

第一次启动会自动执行数据库迁移（Flyway），正常情况无报错。

按 `Ctrl+C` 先停掉，继续下一步。

### 4-3. 让后端在后台常驻运行（systemd 服务）

创建系统服务，开机自动启动、崩溃自动重启：

```bash
sudo nano /etc/systemd/system/eplugger.service
```

粘贴：

```ini
[Unit]
Description=EPlugger Backend
After=network.target mysql.service

[Service]
Type=simple
User=root
EnvironmentFile=/opt/eplugger/env.conf
ExecStart=java -jar /opt/eplugger/eplugger-backend-0.1.0-SNAPSHOT.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

将 `start.sh` 里的 `export` 行整理成 `env.conf` 格式（去掉 `export`，每行一个变量）：

```bash
sudo nano /opt/eplugger/env.conf
```

粘贴（同样替换为你的真实值）：

```env
EPLUGGER_BUSINESS_DB_URL=jdbc:mysql://localhost:3306/eplugger?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
EPLUGGER_BUSINESS_DB_USERNAME=eplugger_user
EPLUGGER_BUSINESS_DB_PASSWORD=你的强密码
JWT_SECRET=请替换成你的随机字符串至少32字符长度abc123xyz
CORS_ALLOWED_ORIGINS=http://你的域名或IP
SSO_FRONTEND_CALLBACK_URL=http://你的域名或IP/sso/callback
EPWORK_SSO_SECRET=与epWork对端一致的密钥
UPLOAD_DIR=/var/lib/eplugger/upload
EPWORK_PERSONNEL_SYNC_ENABLED=false
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable eplugger
sudo systemctl start eplugger

# 查看状态，显示 active (running) 即成功
sudo systemctl status eplugger

# 查看日志
sudo journalctl -u eplugger -f
```

## 第五步：构建前端静态文件

> 这步**在开发机**（你的 Windows 电脑）上做。

### 5-1. 配置生产环境变量

在 `frontend` 目录下创建 `.env.production` 文件（与 `.env.development` 同级）：

```
frontend/.env.production
```

文件内容（替换为你的服务器地址）：

```env
VITE_API_BASE_URL=http://你的域名或IP
```

> **为什么要填这个？** 不填的话，前端代码会默认拼接 `:8080` 端口访问后端，部署后请求会发到 `http://你的IP:8080`，而 Nginx 监听的是 80 端口，导致接口失败。

### 5-2. 安装依赖并构建

```bash
cd frontend
npm install
npm run build
```

构建成功后，产物在 `frontend/dist` 目录。

### 5-3. 把 dist 上传到服务器

```bash
# 在开发机执行，把整个 dist 目录上传
scp -r frontend/dist 你的用户名@服务器IP:/var/www/eplugger
```

在服务器上先创建该目录：

```bash
sudo mkdir -p /var/www/eplugger
```

## 第六步：安装并配置 Nginx

### 6-1. 安装 Nginx

```bash
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```

### 6-2. 配置 Nginx

```bash
sudo nano /etc/nginx/sites-available/eplugger
```

粘贴以下内容：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;   # 例如：192.168.1.100 或 app.example.com

    # 文件上传大小限制，与后端保持一致
    client_max_body_size 20M;

    # 前端静态资源
    root /var/www/eplugger;
    index index.html;

    # 后端 API 接口转发
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # SSO 相关接口转发
    location /sso/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 前端单页应用（SPA）路由回退，刷新页面不 404
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

启用该配置并重启 Nginx：

```bash
sudo ln -s /etc/nginx/sites-available/eplugger /etc/nginx/sites-enabled/
sudo nginx -t          # 检查配置语法，显示 ok 才继续
sudo systemctl reload nginx
```

## 第七步：验证部署是否成功

1. 打开浏览器，访问 `http://你的服务器IP或域名`，能看到登录页面说明前端 OK。
2. 登录成功说明后端 API 与数据库连通正常。
3. 发一条动态并带图片，看图片能否正常显示，说明上传功能 OK。

### 常用排查命令

```bash
# 查看后端实时日志
sudo journalctl -u eplugger -f

# 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log

# 检查后端端口是否在监听
sudo ss -tlnp | grep 8080

# 重启后端
sudo systemctl restart eplugger

# 重启 Nginx
sudo systemctl reload nginx
```

## 第八步：更新部署（日后有新版本时）

### 更新后端

1. 在开发机重新构建：`mvn -DskipTests package`
2. 上传新 JAR：`scp backend/target/*.jar 你的用户名@服务器IP:/opt/eplugger/`
3. 重启服务：`sudo systemctl restart eplugger`

### 更新前端

1. 修改代码后在开发机重新构建：`npm run build`
2. 上传新 dist：`scp -r frontend/dist/* 你的用户名@服务器IP:/var/www/eplugger/`
3. Nginx 无需重启，静态文件立即生效。

## 附录：上线安全检查（上线前对照检查）

- [ ] `JWT_SECRET` 已替换为至少 32 字符的随机字符串，不是代码里的默认值
- [ ] `EPWORK_SSO_SECRET` 与 epWork 对端一致
- [ ] MySQL 业务库密码已自定义，非空弱口令
- [ ] `CORS_ALLOWED_ORIGINS` 填写的是你的实际域名/IP，没有使用 `*`
- [ ] `SSO_FRONTEND_CALLBACK_URL` 填写的是正式地址
- [ ] `env.conf` 文件权限设置合适，避免其他用户读取（`sudo chmod 600 /opt/eplugger/env.conf`）
- [ ] 服务器防火墙只开放了 80（和 443），**不要**对外暴露 8080 端口

## 附录：环境变量含义速查

| 变量名 | 是否必填 | 含义 |
|--------|----------|------|
| `EPLUGGER_BUSINESS_DB_URL` | 必填 | 业务数据库 JDBC 地址 |
| `EPLUGGER_BUSINESS_DB_USERNAME` | 必填 | 数据库用户名 |
| `EPLUGGER_BUSINESS_DB_PASSWORD` | 必填 | 数据库密码 |
| `JWT_SECRET` | 必填 | JWT 签名密钥，≥32 字符 |
| `CORS_ALLOWED_ORIGINS` | 必填 | 允许访问的前端地址（你的域名/IP） |
| `SSO_FRONTEND_CALLBACK_URL` | 必填 | epWork SSO 成功后跳回的前端地址 |
| `EPWORK_SSO_SECRET` | 必填 | epWork 外链验签密钥 |
| `UPLOAD_DIR` | 必填 | 上传文件存储路径 |
| `EPWORK_PERSONNEL_SYNC_ENABLED` | 必填 | 是否同步人员库：`true` / `false` |
| `EPWORK_PERSONNEL_DB_URL` | 同步时必填 | 人员库 JDBC 地址 |
| `EPWORK_PERSONNEL_DB_USERNAME` | 同步时必填 | 人员库用户名 |
| `EPWORK_PERSONNEL_DB_PASSWORD` | 同步时必填 | 人员库密码 |
| `VITE_API_BASE_URL`（前端构建） | 必填 | 前端调接口的根地址（服务器 IP 或域名） |
